API Documentation

Integrate STT into your applications.

1. Request Details

POST /transcribe

Content-Type: multipart/form-data

Required Parameters:

  • file: The audio file binary (mp3, wav, m4a, ogg, flac). Max size: 25MB.

Optional Parameters:

  • language: string (e.g., en, es). Helps AI accuracy.
  • webhook_url: string (URL for async result delivery).

2. Response & Webhooks

Immediate Response (202 Accepted):

{
  "task_id": "uuid-v4",
  "status": "pending",
  "message": "Audio is processing..."
}

Webhook Payload (Sent when complete):

{
  "event": "transcription.completed",
  "task_id": "uuid-v4",
  "data": {
    "duration_seconds": 120.5,
    "transcript": "Hello and welcome...",
    "language": "en"
  }
}

3. Code Snippets

cURL:

curl -X POST "http://127.0.0.1:8000/transcribe" \
  -H "x-api-key: YOUR_API_KEY" \
  -F "file=@/path/to/audio.mp3" \
  -F "language=en"

Node.js (Fetch):

const formData = new FormData();
formData.append('file', fs.createReadStream('audio.mp3'));
formData.append('language', 'en');

const response = await fetch('http://127.0.0.1:8000/transcribe', {
  method: 'POST',
  headers: { 'x-api-key': 'YOUR_API_KEY' },
  body: formData
});
const data = await response.json();
console.log(data);
